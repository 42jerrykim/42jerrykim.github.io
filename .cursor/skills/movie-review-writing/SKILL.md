# 영화 리뷰 작성 에이전트

영화를 웹 검색으로 심층 조사한 뒤, 스포일러 포함 상세 줄거리·구조 분석·캐릭터/상징/영상미 분석까지 아우르는 전문 리뷰를 작성한다.

> **기존 규칙과의 관계**: `content/collection/Movies/.cursor/rules/movie-review-writing-rules.mdc`가 Front Matter 템플릿·폴더명·본문 섹션 구조를 정의한다. **본문·추천 작품·비교 언급에서 쓰는 외국 영화 한글 제목**(공식 개봉명·의역·통용 음차, `《영문 (한글)》` 형식 등)은 동 파일의 **「본문·추천 작품의 한국어 제목」** 절을 따른다. 이 스킬은 **리서치 워크플로우**와 **심층 분석 방법론**에 집중한다.

---

## Phase 1: 사전 조사 (Research)

영화 이름을 받으면, 아래 순서대로 최소 **8개 이상의 출처**에서 정보를 수집한다.

### Step 1: WebSearch로 광범위 정보 수집

다음 검색을 **병렬로** 실행한다:

```
검색 쿼리 세트:
1. "{영화명} {연도} full plot summary spoilers"
2. "{영화명} {연도} 줄거리 전체 스포일러"
3. "{영화명} {연도} film analysis themes symbolism"
4. "{영화명} {연도} behind the scenes making of production"
5. "{영화명} {연도} cinematography visual style analysis"
6. "{영화명} {연도} soundtrack score music analysis"
7. "{영화명} {연도} cast characters analysis"
8. "{영화명} {연도} review rotten tomatoes metacritic"
```

### Step 2: Browser MCP로 상세 탐색

WebSearch 결과에서 유용한 URL을 선별하여 `browser_navigate`로 접근한다.

**우선순위 소스**:
1. **Wikipedia** — 줄거리 전체 요약, 제작 배경, 흥행 정보
2. **IMDb** — 캐스트/스태프, 트리비아, 평점
3. **Rotten Tomatoes / Metacritic** — 비평 종합 점수
4. **나무위키 / 네이버 영화** — 한국어 정보, 국내 평점
5. **전문 영화 평론 사이트** — Roger Ebert, IndieWire, Collider 등
6. **유튜브 영상 에세이** (제목/설명만) — 분석 관점 아이디어

**Browser MCP 워크플로우**:
```
1. browser_navigate → 대상 URL 로드
2. browser_snapshot → 페이지 구조 파악
3. 필요한 섹션 텍스트 추출 (browser_click으로 탭/아코디언 열기)
4. 다음 소스로 이동
```

> **중요**: browser_lock/unlock 순서를 지킨다. navigate → lock → 상호작용 → unlock.

### Step 3: 정보 정리

수집한 정보를 다음 카테고리로 분류한다:

| 카테고리 | 수집 내용 |
|---------|----------|
| 기본 정보 | 제목(영·한), **한국 개봉/배급 공식 한글 제목**, 감독, 캐스트, 장르, 상영시간, 개봉일, 제작사, 제작비 |
| 줄거리 | 시간순 전체 플롯 (크로스체크 필수) |
| 평점/흥행 | RT, IMDb, Metacritic, 네이버, CGV 등 |
| 제작 비화 | 캐스팅 비화, 촬영 기법, 특수효과, 제작 과정 |
| 비평 관점 | 전문 비평가 의견 요약, 찬반 논점 |
| 상징/주제 | 핵심 테마, 메타포, 사회적 맥락 |
| 영상/음악 | 촬영 스타일, 색감, 사운드트랙 정보 |

## Phase 2: 포스트 생성

### Step 4: 폴더 및 Front Matter 생성

1. 터미널에서 오늘 날짜 확인: `Get-Date -Format "yyyy-MM-dd"`
2. `movie-review-writing-rules.mdc`의 폴더명 규칙과 Front Matter 템플릿을 따른다
3. `draft: true` 필수
4. tags 50개 이상 (영어+한글 쌍)
5. 대상 영화 및 **추천·인용할 타 작품**의 한글 제목을 네이버 영화·KOFIC·한글 위키 등에서 확인해, 규칙의 「본문·추천 작품의 한국어 제목」에 맞게 메모해 둔다.

### Step 5: 본문 작성

`movie-review-writing-rules.mdc`의 본문 구조를 기본으로 하되, 아래 **심층 분석 섹션**을 추가한다.

- **추천 작품**·본문 속 타작 언급은 `《영문원제 (한글 개봉명)》(연도)` 형식을 기본으로 하고, 임의 음차(예: October → 오큰토버)를 쓰지 않는다.

## Phase 3: 심층 분석 프레임워크

기존 규칙의 기본 구조(개요 → 구조 분석 → 줄거리 → 캐릭터 → 영상미/음악 → 종합 평가) 위에 다음을 강화한다.

### 3-A. 상징과 메타포 분석

줄거리 작성 후, 영화 속 상징적 요소를 별도 섹션으로 분석한다:

```markdown
## 상징과 메타포 분석

### 시각적 상징
- **반복 이미지/색상**: 영화에서 반복되는 시각적 모티프와 그 의미
- **공간의 상징성**: 장소/배경이 상징하는 바

### 서사적 메타포
- **캐릭터 이름/설정의 의미**: 이름, 직업, 관계에 담긴 의도
- **플롯 구조 자체의 메타포**: 서사 구조가 전달하는 메시지

### 핵심 주제 의식
- **표면 주제**: 관객이 직관적으로 느끼는 주제
- **심층 주제**: 비평적 분석을 통해 드러나는 주제
- **사회적/문화적 맥락**: 개봉 당시 시대상과의 연결
```

### 3-B. 제작 비화와 영화사적 의의

```markdown
## 제작 비화

### 기획과 제작 과정
- 영화의 기획 배경, 원작(있는 경우), 감독의 의도
- 캐스팅 과정의 흥미로운 에피소드
- 제작 중 겪은 도전과 해결

### 기술적 혁신
- 촬영/VFX/사운드 등 기술적으로 주목할 점
- 이전 작품 대비 발전한 기법

### 영화사적 위치
- 해당 장르/감독 필모그래피에서의 위치
- 후대 영화에 미친 영향
- 박스오피스/비평/수상 이력
```

### 3-C. 감독론 분석

```markdown
## 감독 분석

### 감독의 스타일과 특징
- 이 감독의 일관된 연출 경향 (시각 스타일, 서사 구조, 테마)
- 이전 작품과의 연결점

### 이 영화에서의 연출 특징
- 특기할 만한 연출 선택과 그 효과
- 다른 감독이었다면 달랐을 지점
```

## Phase 4: 품질 검증

### Step 6: 최종 체크리스트

기존 `movie-review-writing-rules.mdc`의 체크리스트에 추가:

- [ ] 8개 이상 출처에서 정보를 수집했는가?
- [ ] 줄거리를 2개 이상 출처로 크로스체크했는가?
- [ ] 상징/메타포 분석이 구체적 장면 근거를 포함하는가?
- [ ] 제작 비화가 출처 기반인가 (추측 아닌)?
- [ ] 감독 분석이 필모그래피 맥락을 포함하는가?
- [ ] 참고 문헌이 5개 이상이고 모든 URL이 접근 가능한가?
- [ ] 본문 링크를 HTTP 요청으로 검증했는가?
- [ ] 추천 작품·인용 작품의 한글 제목이 규칙 「본문·추천 작품의 한국어 제목」(공식명·의역·통용 음차)을 만족하는가?

## 빠른 시작 예시

사용자가 "인터스텔라 리뷰 작성해줘"라고 요청하면:

1. **WebSearch 8회** 병렬 실행 (줄거리, 분석, 제작 비화, 영상, 음악, 캐스트, 평점, 상징)
2. **Browser MCP**로 Wikipedia, IMDb, RT 상세 탐색
3. 정보 분류 → 크로스체크
4. 날짜 확인 → 폴더 생성 → Front Matter 작성
5. 본문: 도입 → 개요 → Act 5 구조 분석 → 전체 줄거리 → 캐릭터 → **상징/메타포** → **제작 비화** → **감독론** → 영상미/음악 → 종합 평가 → 참고 문헌
6. 체크리스트 검증
