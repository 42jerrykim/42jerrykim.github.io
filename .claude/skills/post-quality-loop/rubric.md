# 게시물 품질 채점 루브릭 (Quality Rubric)

이 문서는 블로그 글 품질을 **가중 100점 + 점수 앵커**로 채점하는 기준표다. 새 기준을 만든 것이 아니라 기존 스킬의 규칙을 **점수화**한 것이다 — 근거 스킬을 항목마다 명시한다. `post-quality-loop` 루프의 채점자(`quality-critic`)가 이 표를 단일 기준으로 사용한다.

## 튜닝 파라미터

| 파라미터 | 기본값 | 의미 |
|----------|--------|------|
| 통과 임계값 | **총점 ≥ 90 AND 치명결함 0** | 이 조건을 만족하면 루프 종료(Record) |
| 최대 반복 | **3** | 미달이 3회 반복되면 사람에게 에스컬레이션 |

> 임계값·반복수는 운영하며 조정할 수 있다. 변경 시 이 표만 고치면 루프 전체에 반영된다.

---

## 채점 항목·가중치 (합 100)

| # | 항목 | 배점 | 근거 스킬 |
|---|------|-----:|-----------|
| 1 | 정확성·사실 근거 | 20 | `educational-content-writing` §4.4, `collection-writing-standards` §1.3 |
| 2 | 깊이·전문가 요소 | 18 | `educational-content-writing` §2.4 |
| 3 | 구조·문단 중심 | 15 | `collection-writing-standards` §2.1, `educational-content-writing` §2.0 |
| 4 | 안티패딩 | 15 | `educational-content-writing` §2.5 |
| 5 | 코드·예제 품질 | 12 | `educational-content-writing` §5.2 |
| 6 | 시각요소(Mermaid·표) | 8 | `rules-that-must-be-followed` §5 |
| 7 | 메타·링크·frontmatter | 12 | `rules-that-must-be-followed` §1·§3 |

각 항목은 **0 / 40 / 70 / 100** 앵커로 채점하고, 배점에 비례해 환산한다(예: 항목 1을 70점 앵커로 보면 `20 × 0.7 = 14`점).

---

## 점수 앵커

### 1. 정확성·사실 근거 (20점)

| 앵커 | 상태 |
|-----:|------|
| 100 | 연도·인물·수치·인용이 출처와 함께 검증됨. 불확실한 기술 사실은 "구현 정의"/"플랫폼에 따라"로 표기. "정량적/측정" 표방 시 실제 수치(플랫폼·플래그) 또는 실행 가능한 벤치 포함 |
| 70 | 대부분 정확하나 일부 수치·연도에 출처가 없거나 검증 흔적이 약함 |
| 40 | 검증 없는 단정("수 배 빠르다", "항상 가장 빠르다")이 다수 |
| 0 | 환각된 사실·출처(존재하지 않는 책/논문/연도) 또는 명백한 오류 → **치명결함** |

### 2. 깊이·전문가 요소 (18점)

| 앵커 | 상태 |
|-----:|------|
| 100 | 학습 성과 목표(평가 기준), 언제 쓰고/피할지 판단 기준, 비판적 시각(한계·트레이드오프·대안)을 모두 갖춤 |
| 70 | 셋 중 둘 충족. 판단 기준 또는 비판적 시각 중 하나가 얕음 |
| 40 | 정의·설명 위주로 암기형. 판단·적용·비판 요소 거의 없음 |
| 0 | 표면적 나열만, 전문가 독자에게 줄 가치 없음 |

### 3. 구조·문단 중심 (15점)

| 앵커 | 상태 |
|-----:|------|
| 100 | 핵심 설명이 문단으로 서술돼 리스트·표·코드를 빼도 요지가 전달됨. 이론→예제→주의 순. 문단이 본문의 ≥40% |
| 70 | 대체로 문단 중심이나 일부 절이 리스트·코드에만 의존 |
| 40 | 설명이 리스트·표 위주, 문단이 빈약. 예제가 설명보다 먼저 나오는 역순 다수 |
| 0 | 코드/리스트 덤프, 연결 문단 거의 없음 |

### 4. 안티패딩 (15점)

| 앵커 | 상태 |
|-----:|------|
| 100 | 닫는 절(요약·FAQ·체크리스트·네비)이 종류별 1개. 답 없는 번호 덤프·중복 요약 없음. 모든 절이 학습 가치를 가짐 |
| 70 | 사소한 중복 1~2곳, 핵심엔 영향 없음 |
| 40 | 중복 요약 절 또는 답 없는 긴 리스트(질문 24/메모 20류) 존재 |
| 0 | 분량 채우기용 반복·필러가 본문 상당 부분. "이 절을 지워도 잃는 게 없는" 블록 다수 |

> 작성자용 메타블록("게시 전 자가 점검", "평가 기준 재확인: 충족") 본문 잔존은 점수와 무관하게 **치명결함**.

### 5. 코드·예제 품질 (12점)

| 앵커 | 상태 |
|-----:|------|
| 100 | 핵심 예제가 `#include`·타입 정의 갖춰 그대로 컴파일/실행 가능, 블록 앞 개념 문단 존재 |
| 70 | 컴파일되나 일부 로직 생략 또는 코드-문단 균형 부족 |
| 40 | 의사코드인데 `text` 미표기, 코드만 연속 |
| 0 | `/* 처리 로직 */` 주석뿐·미정의 타입(`T`,`vec`)으로 핵심 대체 |

> 코드가 거의 없는 글(예: Vocabulary, 영화 리뷰)은 이 항목을 **N/A**로 두고 배점을 항목 2·3에 비례 재분배한다.

### 6. 시각요소 (Mermaid·표) (8점)

| 앵커 | 상태 |
|-----:|------|
| 100 | 구조/흐름/비교에 Mermaid 또는 표를 적절히 사용. Mermaid 라벨 따옴표·`</br>`·노드 ID 규칙 준수 |
| 70 | 시각요소는 있으나 일부 문법 경고 소지(따옴표 누락 등) |
| 40 | 시각요소가 거의 없어 복잡한 구조가 글로만 설명됨 |
| 0 | Mermaid 문법 오류로 렌더 실패 가능 → **치명결함** |

### 7. 메타·링크·frontmatter (12점)

| 앵커 | 상태 |
|-----:|------|
| 100 | title ≤70자, description ~150자(120~170), tags ≥50(`data/tags.yaml` 기반), `draft` 필드 존재, 모든 외부 링크 접근 가능, 내부 링크 `/post/...` 형식 |
| 70 | 위 중 1개 경미한 미달(예: tags 45개, description 110자) |
| 40 | 2개 이상 미달 또는 일부 외부 링크 미검증 |
| 0 | 깨진 외부 링크(404/5xx), 존재하지 않는 파일·이미지 참조, frontmatter 파싱 불가 → **치명결함** |

> **`draft` 필드 판정**: 신규 초안은 `draft: true`가 있어야 한다(없으면 결함). 이미 게시된 글은 `draft` 생략(= 게시) 또는 `draft: false`가 정상이며 결함이 아니다. 즉 "draft 상태가 의도와 어긋날 때"만 감점한다 — 게시글 전부를 치명결함으로 보지 않는다.

---

## 치명결함 (Critical) — 점수와 무관하게 게시 차단

총점이 임계값을 넘어도 아래 중 하나라도 있으면 **통과 불가**(에스컬레이션 또는 Revise 대상):

1. 깨진 외부 링크(404/5xx) 또는 저장소에 없는 파일 참조
2. Mermaid 문법 오류로 렌더/빌드 실패 가능
3. 작성자용 메타블록("게시 전 자가 점검" 등) 본문 잔존
4. 환각된 사실·출처(검증 안 된 연도·수치·인용)
5. frontmatter 파싱 불가, 또는 `draft` 상태가 의도와 모순(신규 초안인데 `draft: true` 없음 등) — 게시글의 `draft` 생략 자체는 정상

## 컬렉션 보정

대상 글이 컬렉션에 속하면, 해당 컬렉션 스킬의 체크리스트를 **항목 2(깊이)·3(구조) 채점에 함께 반영**한다.

| 컬렉션 폴더 | 보정 기준 스킬 |
|-------------|----------------|
| `Algorithm/` | `algorithm-post-writing` (문제 정보·복잡도 표·코너 케이스) |
| `Vocabulary/` | `vocabulary-post-writing` (정의·콜로케이션·유의어·예문) |
| `Movies/` | `movie-review-writing` (Act 5 구조·캐릭터·상징) |
| `TV-Show/` | `tv-series-review-writing` (출처 10+·Act 5·인물 분석) |
| `bashshell/` | `bashshell-post-writing` |
| `cmd/` | `cmd-post-writing` |
| 교육 시리즈 (`optimization-*`, `design-patterns`, `cleanarchitecture` 등) | `educational-content-writing` + `collection-writing-standards` (00 챕터 필수요소·레벨별 경로·Broken→Fixed→Verification·문단 비율 ≥40%·컴파일 가능 코드·수치 근거) |

해당 컬렉션 체크리스트의 필수 섹션이 빠졌으면 항목 2·3에서 감점하고, 결함 목록에 "어느 섹션이 빠졌는지"를 적는다.
